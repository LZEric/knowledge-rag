# 核心思想

多路由 + 动态知识库

# 过程

1. 文档解析得到文本数据
2. 将文本数据分块（包含页码信息）
3. 向量化放入向量数据库
4. 基于问题路由到向量数据库对应部分（混合检索 关键词+向量）
5. 使用LLM进行Rerank
6. 将Rerank的结果、上下文和用户的提示词进行拼接

# 技术点

1. 文档中大表格的处理（保证语义的连贯性）—— 以html格式来喂给llm，而不是md，大模型理解的更好且适配更多的表格格式（Docling英文，如果中文的pdf则推荐使用MinerU）
2. 混合检索（BM25 + 向量）
3. 路由
4. 思维链（CoT）
5. 指令细化（可解释的边缘情况；没有答案怎么处理）
