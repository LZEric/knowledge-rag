# 可持续 AI 编程

使用 AI 辅助进行长期开发时，保持高质量、可维护代码库的指南。

## 核心原则

### 1. 代码即契约

将代码视为当前和未来开发者（包括 AI）之间的沟通媒介。

**指南：**
- 编写能自我解释意图的代码，而不仅仅是实现细节
- 优先考虑清晰而非技巧性
- 文档化假设和设计决策
- 保持依赖显式化

### 2. AI 协同而非替代

AI 辅助应增强人类判断，而非取代它。

**指南：**
- 批判性审查 AI 建议
- 接受前理解代码逻辑
- 优先考虑增量变更而非大规模重写
- 保持架构意图清晰

### 3. 减法优于加法

长期可维护性需要移除复杂性，而不仅仅是添加功能。

**指南：**
- 及时删除无效代码
- 弃用过时功能
- 使用 AI 替换时进行简化
- 质疑新抽象的必要性

### 4. 上下文保留

AI 需要上下文才能有效工作。让代码库状态易于理解。

**指南：**
- 保持 README 更新
- 文档化架构决策（ADR）
- 维持一致的项目结构
- 注释非显而易见的决策

## 使用场景

当用户询问以下内容时使用本技能：
- 搭建 AI 辅助开发工作流
- 使用 AI 辅助维护代码质量
- 项目长期可持续性
- 跨团队扩展 AI 辅助
- 创建可维护的 AI 生成代码
- 平衡 AI 速度与人工监督

## 关键实践

### AI 协作式代码审查

1. **提交前**：AI 建议变更 → 人工审查意图
2. **提交时**：人工对变更负责
3. **提交后**：AI 监控一致性
4. **重构时**：人工识别需求，AI 建议改进

### 测试策略

- 单元测试：AI 生成，人工验证覆盖率
- 集成测试：人工设计，AI 实现
- 端到端测试：人工指定，AI 执行
- 性能测试：AI 建议，人工验证

### 文档

- 代码注释：AI 建议，人工优化
- API 文档：AI 生成，人工上下文化
- 架构文档：人工主导，AI 协助
- 教程：人工编写，AI 格式化

## 常见陷阱

| 陷阱 | 预防措施 |
|------|----------|
| AI 代码纠缠 | 定期代码库审计 |
| 隐形 AI | 文档化所有 AI 辅助变更 |
| 上下文衰减 | 自动化文档更新 |
| 质量漂移 | CI 检查代码质量 |
